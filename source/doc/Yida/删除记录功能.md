# 删除管理功能

本文档介绍如何配置后端接口并实现数据删除功能，包含接口配置、弹窗交互及核心逻辑代码。

## 配置后端接口

在项目的接口配置文件中，添加删除数据的请求地址。请根据实际应用类型替换 `appType` 路径变量。

| 配置项 | 请求地址 |
| --- | --- |
| `deleteData` | `/${window.pageConfig.appType}/v1/form/deleteFormData.json` |

## 实现删除交互

### 显示删除确认弹窗

调用以下方法展示删除对话框，并自动将选中记录的实例ID填充到弹窗输入框中。

```jsx
// 删除活动弹窗
export function onDeleteActivityDialog(rowData) {
  this.$('dialog_l9njsj8t').show(() => {
    this.$('textField_los2eejb').setValue(rowData.formInstId);
  });
}
```

### 确认删除操作

用户点击确认后，执行以下核心步骤：

- **开启加载状态**：避免重复提交
- **调用删除接口**：传递表单实例ID
- **处理响应结果**：
  - **成功**：关闭弹窗、提示成功、刷新数据列表
  - **失败**：显示错误信息

**完整代码示例：**

```jsx
// 确认删除活动
export function onDeleteActivityOk() {
  // 1. 开启对话框加载状态
  this.$('dialog_l9njsj8t').set('confirmState', 'LOADING');

  // 2. 调用删除接口
  this.dataSourceMap.deleteData.load({
    formInstId: this.$('textField_los2eejb').getValue(),
  })
    .then(() => {
      this.$('dialog_l9njsj8t').set('confirmState', 'NORMAL');
      this.$('dialog_l9njsj8t').hide();
      this.utils.toast({
        title: '删除成功',
        type: 'success',
      });
      setTimeout(() => {
        this.setState({ tableIsLoading: true });
        this.getData();
      }, 1000);
    })
    .catch(({ message }) => {
      this.utils.toast({
        title: message,
        type: 'error',
      });
      this.$('dialog_l9njsj8t').set('confirmState', 'NORMAL');
    });
}
```

> **注意事项**
>
> - 接口返回的错误信息应直接通过 `message` 字段向用户展示
> - 刷新数据前设置 `tableIsLoading` 状态，可提升列表重新加载时的用户体验
